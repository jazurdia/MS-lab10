  • En qu situaciones es preferible usar el Muestreador de Gibbs sobre Metropolis-Hastings?

    El muestreador de Gibbs es preferible cuando las distribuciones condicionales de los parmetros son fciles de muestrear directamente. Es decir, cuando los parmetros de inters tienen distribuciones condicionales conjugadas o simplificadas, lo que facilita el proceso de muestreo y no se tiene que rechazar propuestas. Por otro lado, el Algoritmo de Metropolis-Hastings se usa cuando las condicionales no son fciles de muestrear directamente y se necesita una distribucin de propuesta, esto implica tiempos de convergencia ms largos y una tasa de aceptacin menor.   • Cmo afecta la eleccin de la distribucin de propuesta en la eficiencia del Algoritmo de Metropolis-Hasting y cmo afecta la tasa de aceptacin de las muestras?

    La eleccin de la distribucin de propuesta en el algoritmo de Metropolis-Hastings afecta directamente su eficiencia y la tasa de aceptacin de las muestras. Una distribucin de propuesta adecuada facilita una exploracin eficiente del espacio de parmetros, lo que puede reducir la autocorrelacin entre las muestras y acelerar la convergencia hacia la distribucin objetivo. Si la distribucin de propuesta es demasiado amplia, las propuestas pueden ser frecuentemente rechazadas, lo que disminuye la tasa de aceptacin y ralentiza el muestreo. Por otro lado, una distribucin demasiado estrecha puede resultar en una alta tasa de aceptacin pero en una exploracin ineficiente del espacio, ya que las muestras sucesivas sern muy similares entre s. Por lo tanto, es esencial seleccionar o adaptar la distribucin de propuesta para lograr un equilibrio ptimo entre la tasa de aceptacin y la eficiencia del muestreo. import numpy as np
import matplotlib.pyplot as plt
from scipy.stats import norm

# Definimos la distribucin objetivo: distribucin normal estndar N(0,1)
def target_distribution(x):
    return norm.pdf(x, loc=0, scale=1)

# Implementacin del algoritmo de Metropolis-Hastings
def metropolis_hastings(target_pdf, proposal_range, num_samples, initial_value):
    samples = [initial_value]
    current_x = initial_value

    for i in range(num_samples):
        # Proponemos un nuevo valor utilizando una distribucin uniforme en [current_x - 1, current_x + 1]
        proposed_x = current_x + np.random.uniform(-proposal_range, proposal_range)

        # Calculamos la probabilidad de aceptacin
        acceptance_ratio = target_pdf(proposed_x) / target_pdf(current_x)
        acceptance_probability = min(1, acceptance_ratio)

        # Generamos un nmero aleatorio para decidir si aceptamos la propuesta
        u = np.random.uniform(0, 1)

        if u < acceptance_probability:
            current_x = proposed_x  # Aceptamos la propuesta
        # Si no se acepta, mantenemos el valor actual

        samples.append(current_x)

    return np.array(samples)

# Parmetros
proposal_range = 1  # Rango de la distribucin de propuesta uniforme [-1, 1]
num_samples = 10000  # Nmero de iteraciones
initial_value = 0  # Valor inicial

# Generamos las muestras utilizando el algoritmo de Metropolis-Hastings
samples = metropolis_hastings(target_distribution, proposal_range, num_samples, initial_value)

# Generamos valores para la distribucin normal verdadera
x = np.linspace(-4, 4, 1000)
y = norm.pdf(x, loc=0, scale=1)

# Graficamos el histograma de las muestras y la distribucin normal
plt.figure(figsize=(10, 6))
plt.hist(samples, bins=50, density=True, alpha=0.6, color='skyblue', label='Muestras de Metropolis-Hastings')
plt.plot(x, y, 'r', label='Distribucin Normal N(0,1)')
plt.title('Aproximacin de N(0,1) usando Metropolis-Hastings')
plt.xlabel('Valor')
plt.ylabel('Densidad de probabilidad')
plt.legend()
plt.show()
